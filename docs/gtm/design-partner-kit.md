## The deal (what we ask / what they get)

**Ask:** run ≥2 real deliberations with a real group, plus one 30-minute
debrief call. **Give:** free forever anyway, but partners get direct influence
on what gets built next, first access to the hosted tier when it exists, and
(for Marcuses) a named thank-you in the launch post if they want it.

## Onboarding steps (send after a yes)

1. Open https://cjander18.github.io/synergon/app.html → "Load the demo
   workflow" — 2 minutes to see the whole loop.
2. Create a real workflow for a decision you actually need input on (retro
   themes, quarter risks, prioritization). 3–8 participants works best.
3. Issue the round; send each person their link and password **through
   different channels** (the app explains this).
4. When responses are in: consolidate, then run a second round — clarify or
   rank. **The second round matters most to us**: that's the multi-round cost
   nobody has measured.
5. Export a backup of the workflow when you finish (button at the top).
6. Anything confusing or broken, at any point: synergonlabs@gmail.com — raw,
   unfiltered notes are exactly what we want.

## Debrief script (30 min)

1. Walk me through the session — where did you stall? (10 min, let them talk)
2. What did the *contributors* say? Any refuse, any phone problems, any
   password friction? (5)
3. The redistribution question: round 2 required sending fresh links to
   everyone. Honestly — how close was that to "not worth it"? (5)
4. The output: did the consolidated/ranked result change your decision, or
   confirm it? Did the group trust it? (5)
5. **The killer question:** "Would you run your next real decision through
   this?" If hesitation: "What one change flips that to yes?" (3)
6. "Would you pay $10/month for links that just work — no copy/paste?" (tally
   for the Tier-1 trigger; do not promise anything) (2)

Log verbatims in the learning ledger the same day. Rule of three applies
([launch-week skill](../../.claude/skills/launch-week/SKILL.md)).

## Partner tracking

Keep a table (spreadsheet or here) with: name · persona type · org size ·
sessions run · rounds per session · debrief date · killer-question answer ·
would-pay-for-relay · standout verbatim. The Phase-2 gate reads from this:
**≥70% complete a multi-round workflow AND ≥50% answer the killer question
yes** — otherwise pause the public launch and reconsider pulling the relay
forward ([launch-plan.md](launch-plan.md)).
